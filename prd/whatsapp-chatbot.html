Vedantu · PRD Shipped

WhatsApp Conditional Chatbot for Learner Re-engagement

TL;DR

Build a conditional-logic chatbot on WhatsApp to re-engage lapsed Vedantu learners, branching conversation flows based on drop-off reason, course type, and engagement history, to recover dormant users and increase inbound sales leads from the existing base.

Problem

~40% of enrolled learners go dormant within 30 days of purchase. Email re-engagement campaigns have <12% open rates and are hitting deliverability limits. The sales team manually calls lapsed users, an expensive and unscalable approach with inconsistent messaging. WhatsApp has 95%+ open rates among our user base but we have no automated presence on the channel. We're leaving significant re-engagement revenue and referral volume on the table.

Goals

✓Re-engage dormant learners at scale via WhatsApp without adding headcount
✓Route high-intent users to a sales agent or course upgrade flow
✓Personalise message branching by course type and drop-off signal
✓Measure conversion from chatbot interaction to session re-entry

Non-goals

×AI-generated freeform responses: this is conditional logic only, no LLM
×New user acquisition via WhatsApp (out of scope for v1)
×Payment collection through the chatbot flow
P0 Trigger outbound WhatsApp message to users who haven't logged in for 7+ days. Segmented by course type (live / recorded / test prep).
P0 Conditional branch tree: 3 levels deep, branching on user reply (e.g. "Too busy" → schedule reminder; "Lost motivation" → send progress summary; "Found it difficult" → route to tutor). Max 5 reply options per node.
P0 Opt-out handling: any user replying "STOP" is immediately removed from all future chatbot flows and flagged in CRM. GDPR/DPDP compliant.
P1 High-intent detection: user who selects "I want to upgrade" or "Talk to someone" triggers real-time handoff to sales queue in CRM with conversation context attached.
P1 Rate limit: max 1 outbound message per user per 14 days. No user receives more than 3 chatbot sequences per quarter.
P2 A/B test framework for message copy: two variants per campaign, winner auto-promoted after 500 impressions.

Success metrics

+15pp user engagement rate vs. email baseline
+5pp inbound sales leads from re-engaged cohort
Opt-out rate < 8% per campaign
Sales handoff-to-close rate ≥ email channel benchmark

Open questions

WhatsApp Business API provider: evaluate Gupshup vs. Twilio vs. direct Meta integration. Cost and template approval time differ significantly.
Should dormancy threshold be 7 days (more aggressive) or 14 days (lower opt-out risk)? Run sensitivity analysis on historical data first.
Who owns the flow editor post-launch: CRM team or growth PM? Need RACI before handoff.
